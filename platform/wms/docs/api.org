* Overview

The Warehouse Management System (WMS) service handles inventory management,
order allocation, fulfillment workflows, shipping integration, and returns
processing. It is an event-driven service consuming order events from Kafka
and publishing fulfillment events.

** Trace Propagation

WMS is a downstream consumer in the trace flow:

#+begin_src text
Order Service --[Kafka: order.created]--> WMS --[Kafka: order.shipped]--> Notifications
                     ^                          ^
                     |                          |
               traceparent in              traceparent preserved
               message headers             through async flow
#+end_src

All async operations MUST preserve trace context. This includes:
- Kafka message headers (traceparent, tracestate)
- Outbox table rows (traceparent column)
- External API calls (ShipStation webhook context)

*** Trace Propagation to ShipStation

ShipStation webhooks do not support trace context headers. We handle this via:

1. Store traceparent in fulfillment record before API call
2. Include traceparent in ShipStation order metadata (~customField1~)
3. On webhook receipt, extract from metadata and continue trace

#+begin_src clojure
;; Outbound request to ShipStation
(defn create-shipstation-order
  [{:keys [fulfillment traceparent]}]
  (let [child-ctx (trace/create-child traceparent)]
    {:orderId (:order_id fulfillment)
     :orderNumber (:order_id fulfillment)
     :customField1 (trace/format child-ctx)  ;; preserve trace
     :shipTo (format-address (:shipping_address fulfillment))
     :items (map format-item (:items fulfillment))
     ;; ... rest of ShipStation order
     }))

;; Webhook handler
(defn handle-shipstation-webhook
  [webhook-payload]
  (let [traceparent (get-in webhook-payload [:resource :customField1])
        ctx (trace/continue-or-start {"traceparent" traceparent} false)]
    (trace/with-span ctx "shipstation.webhook"
      ;; process webhook
      )))
#+end_src

* Trace Propagation Patterns

** Kafka Consumer Pattern

#+begin_src clojure
(defn handle-order-created
  [message]
  (let [headers (kafka/headers message)
        traceparent (get headers "traceparent")
        ctx (trace/continue-or-start {"traceparent" traceparent} false)]
    (trace/with-span ctx "wms.handle-order-created"
      (let [order (:value message)
            allocation (allocate-order ctx order)]
        (publish-with-trace ctx "fulfillment.allocated" allocation)))))

(defn publish-with-trace
  [ctx topic event]
  (let [child-ctx (trace/create-child ctx)
        headers {"traceparent" (trace/format child-ctx)}]
    (kafka/send! {:topic topic
                  :headers headers
                  :value event})))
#+end_src

** Outbox Pattern for Transactional Events

#+begin_src clojure
(defn allocate-order-transactionally
  [ctx order]
  (jdbc/with-transaction [tx db-spec]
    ;; Reserve inventory
    (inventory/reserve! tx (:items order))

    ;; Create allocation record
    (let [allocation (allocations/create! tx order)]

      ;; Write to outbox within same transaction
      (outbox/insert! tx
        {:event_type "fulfillment.allocated"
         :aggregate_id (:allocation_id allocation)
         :payload allocation
         :traceparent (trace/format ctx)})

      allocation)))

;; Separate process drains outbox to Kafka
(defn drain-outbox
  []
  (doseq [row (outbox/pending-events db-spec)]
    (let [ctx (trace/parse (:traceparent row))]
      (trace/with-span ctx "outbox.publish"
        (kafka/send! {:topic (event->topic (:event_type row))
                      :headers {"traceparent" (:traceparent row)}
                      :value (:payload row)})
        (outbox/mark-published! db-spec (:id row))))))
#+end_src

** HTTP Client Pattern (ShipStation)

#+begin_src clojure
(defn call-shipstation
  [ctx endpoint body]
  (let [child-ctx (trace/create-child ctx)]
    (trace/with-span child-ctx "shipstation.api"
      (http/post (str shipstation-base-url endpoint)
        {:headers {"Authorization" (shipstation-auth)
                   "Content-Type" "application/json"
                   ;; ShipStation doesn't use traceparent header,
                   ;; but we record it for correlation
                   "X-Request-Id" (:span_id child-ctx)}
         :body (json/encode body)
         :timeout 30000}))))
#+end_src

* Database Schema

** inventory table

#+begin_src sql
CREATE TABLE inventory (
    sku VARCHAR(100) NOT NULL,
    warehouse_id VARCHAR(50) NOT NULL,
    on_hand INTEGER NOT NULL DEFAULT 0,
    reserved INTEGER NOT NULL DEFAULT 0,
    available INTEGER GENERATED ALWAYS AS (on_hand - reserved) STORED,
    reorder_point INTEGER NOT NULL DEFAULT 0,
    reorder_quantity INTEGER NOT NULL DEFAULT 0,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    PRIMARY KEY (sku, warehouse_id)
);

CREATE INDEX idx_inventory_low_stock ON inventory(warehouse_id, available)
