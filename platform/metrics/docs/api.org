* Overview

The Metrics Platform provides event collection, A/B experiment assignment, and
analytics querying for the Petstore Commerce Platform. All APIs propagate
trace-id for user journey correlation.

** Design Principles

1. *trace-id is the join key* - All events carry traceparent; user journeys
   are reconstructed by trace-id + session-id aggregation
2. *Write path is async* - Events are fire-and-forget to Kafka; queries hit
   pre-aggregated ClickHouse tables
3. *Experiments are stateless* - Assignment is deterministic hashing; no
   server state required
4. *Feature flags drive experiments* - A/B variants map to feature flag
   evaluation

and published to Kafka. Returns immediately after queueing.

*** Event Types

| Type       | Description                          | Required Properties          |
|------------+--------------------------------------+------------------------------|
| impression | Product/content viewed               | product_id, position         |
| click      | User clicked on element              | element_id or product_id     |
| add_to_cart| Item added to cart                   | product_id, quantity         |
| purchase   | Order completed                      | order_id, total              |
| experiment | Experiment exposure                  | experiment_id, variant       |
| error      | Client-side error                    | error_type, message          |

*** Trace Context

The traceparent header is REQUIRED. Events without valid traceparent are rejected.
The trace_id from traceparent is stored with every event for journey correlation.

*** Rate Limits

| Scope          | Limit            | Window |
|----------------+------------------+--------|
| Per session    | 100 events       | 1s     |
| Per IP         | 1000 events      | 1s     |
| Batch size     | 50 events        | -      |

* A/B Experiment Assignment API

** GET /experiments/assign

Get experiment variant assignment for a user/session. Assignment is deterministic
based on hash(experiment_id + bucketing_key) % 100.

*** Request

#+begin_src http
GET /experiments/assign?experiment_id=checkout_flow_v2&bucketing_key=usr_456 HTTP/1.1
Host: metrics.petstore.internal
traceparent: 00-4bf92f3577b34da6a3ce929d0e0e4736-00f067aa0ba902b7-01
#+end_src

*** Response

#+begin_src json
{
  "experiment_id": "checkout_flow_v2",
  "variant": "treatment",
  "bucketing_key": "usr_456",
  "bucket": 73,
  "metadata": {
    "started_at": "2026-05-01T00:00:00Z",
    "allocation": {

*** Assignment Algorithm

#+begin_src clojure
(defn assign-variant
  "Deterministic variant assignment using murmur3 hash.
   Bucket = hash(experiment_id || bucketing_key) mod 100"
  [experiment-id bucketing-key allocation]
  (let [hash-input (str experiment-id ":" bucketing-key)
        bucket     (mod (murmur3-32 hash-input) 100)]
    (some (fn [[variant [lo hi]]]
            (when (<= lo bucket hi) variant))
          allocation)))
#+end_src

* Real-Time Aggregation Patterns

** Kafka Topics

| Topic                     | Partitions | Retention | Description                    |
|---------------------------+------------+-----------+--------------------------------|
| metrics.events.raw        | 16         | 7d        | All raw events                 |
| metrics.events.validated  | 16         | 7d        | Validated events               |
| metrics.aggregations      | 8          | 1d        | Pre-aggregated rollups         |
| metrics.experiments       | 4          | 30d       | Experiment exposures           |

** Aggregator Processing

The aggregator consumes from metrics.events.validated and produces:

1. *Minute windows* - Tumbling windows, written to Redis for real-time dashboards
2. *Hourly rollups* - Written to ClickHouse events_rollup_hourly
3. *Daily rollups* - Triggered by end-of-day, written to events_rollup_daily

** Aggregation Schema

#+begin_src sql
-- ClickHouse table for hourly rollups
CREATE TABLE events_rollup_hourly (
    event_date Date,
    event_hour DateTime,
    event_type LowCardinality(String),
    experiment_id Nullable(String),
    variant Nullable(String),

    -- Counts
    count UInt64,
    unique_sessions AggregateFunction(uniq, String),
    unique_users AggregateFunction(uniq, String),

    -- Product metrics
    product_impressions UInt64,
    product_clicks UInt64,
    add_to_carts UInt64,
    purchases UInt64,
    revenue Decimal(18, 2)
)
ENGINE = SummingMergeTree()
PARTITION BY event_date
ORDER BY (event_date, event_hour, event_type, experiment_id, variant);
#+end_src

** Feature Flag Mapping

Each experiment is backed by a feature flag. The flag controls:

1. Whether the experiment is active
2. User targeting rules (segments, percentages)
3. Kill switch for immediate rollback

| Experiment Field | Feature Flag Field |
|------------------+--------------------|
| experiment_id    | flag metadata      |
| allocation       | rollout percentage |
| targeting        | user segments      |
| start_date       | schedule           |
| end_date         | schedule           |

* Error Handling

** Error Codes

| Code | HTTP Status | Description                    |
|------+-------------+--------------------------------|
| E001 | 400         | Invalid event schema           |
| E002 | 400         | Missing traceparent            |
| E003 | 400         | Invalid traceparent format     |
| E004 | 429         | Rate limit exceeded            |
| E005 | 404         | Experiment not found           |
| E006 | 403         | Insufficient permissions       |
| E007 | 500         | Internal aggregation error     |
