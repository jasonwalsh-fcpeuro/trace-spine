* Summary

The first PR establishes the minimal foundation that all other work depends on.
It ships the wire format spec, the configuration schema, and the ADR substrate.

* Why this scope

1. **L1-wire.org is the contract** — every adapter binds to it; the conformance
   harness derives fixtures from it; ADR-0002 cites it.

2. **ADR substrate is required** — exemptions require ADR ids; spec changes
   require ADR references; establishing the ADR process first avoids retrofitting.

3. **JSON Schema enables validation** — consumer repos can validate their
   `.trace-spine.yaml` before the harness exists.

4. **Nothing else is needed** — the regex, header names, and format are complete
   with just these files. No library code, no harness, no integration.

* Refutation condition (do NOT merge if)

The PR should NOT merge if any of the following are true:

1. **Regex rejects valid W3C traceparent** — the validation regex must accept
   all examples from the W3C Trace Context spec.

2. **Regex accepts invalid traceparent** — the regex must reject:
   - trace-id of all zeros
   - parent-id of all zeros
   - version other than 00
   - wrong field lengths

3. **JSON Schema does not validate canonical example** — the schema must
   validate the example `.trace-spine.yaml` in this PR.

4. **ADR-0002 conflicts with W3C spec** — the ADR must not claim behavior
   that contradicts the W3C Trace Context recommendation.

5. **spec-keeper has not reviewed** — this PR touches spec/; spec-keeper
   sign-off is mandatory.
