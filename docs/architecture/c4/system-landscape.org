* Overview

This document contains all four levels of the C4 model for the Petstore Commerce
Platform — a Clojure/ClojureScript ecommerce system with trace-spine integration.

All diagrams use Mermaid with C4 extensions. The diagrams are the source of truth;
code must conform to these contracts.

** Maintenance Protocol

C4 diagrams MUST be updated when:
1. New service is added
2. Service boundary changes
3. Data flow changes between containers
4. New external system integration

Agents working on platform/* MUST run ~bin/c4-check~ before committing.

** Code Diagram Generation

Each service has a ~docs/generate-code-diagram.clj~ script that:
1. Parses the namespace structure
2. Extracts public function signatures
3. Generates Mermaid class diagrams
4. Outputs to ~docs/code-diagram.org~

Run via: ~clj -M:dev -m docs.generate-code-diagram~

* Trace Propagation Map

How traceparent flows through the system:

#+begin_src mermaid :file trace-flow.png :eval never-export
sequenceDiagram
    participant Browser as SPA (ClojureScript)
    participant API as API Gateway
    participant Order as Order Service
    participant Fraud as Fraud Service
    participant Payment as Payment Service
    participant Kafka as Kafka
    participant WMS as WMS Service

    Browser->>+API: POST /checkout (traceparent: 00-abc...)
    API->>API: trace-spine middleware extracts context
    API->>+Fraud: POST /score (traceparent: 00-abc...-child1)
    Fraud-->>-API: {score: 0.2}
    API->>+Order: POST /orders (traceparent: 00-abc...-child2)
    Order->>+Payment: POST /charge (traceparent: 00-abc...-child3)
    Payment-->>-Order: {status: success}
    Order->>Kafka: order.created (traceparent in header)
    Order-->>-API: {order_id: 123}
    API-->>-Browser: {order_id: 123}

    Kafka->>+WMS: consume order.created (traceparent: 00-abc...)
    WMS->>WMS: Allocate inventory
    WMS->>Kafka: order.shipped (traceparent preserved)
    WMS-->>-Kafka: ack
#+end_src

* Data Flow Summary

| Source | Target | Protocol | Trace Carrier |
|--------|--------|----------|---------------|
| SPA | API Gateway | HTTPS | traceparent header |
| Admin | API Gateway | HTTPS | traceparent header |
| API Gateway | Any Service | HTTP | traceparent header |
| Service | Service | HTTP | traceparent header |
| Service | Kafka | Kafka | traceparent in message headers |
| Service | PostgreSQL | JDBC | span context in outbox table |
| Kafka Consumer | Service | Kafka | traceparent from message headers |
