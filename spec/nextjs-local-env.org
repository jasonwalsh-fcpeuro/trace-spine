* Overview

Full local development environment for a Next.js application instrumented with
trace-spine. Everything runs in Docker Compose. Hooks fire from the host
(Claude Code / opencode) against container logs, test output, and telemetry.

** Multi-stack mirroring

The trace-spine exploration focuses on Clojure/ClojureScript as the primary
substrate, but the harness pattern is stack-agnostic. Mirrored versions of
this environment spec exist (or should be created) for each target stack:

| Stack              | App framework          | PBT library          | Debugger           | Status   |
|--------------------+------------------------+----------------------+--------------------+----------|
| Clojure/CLJS       | Ring + Re-frame        | test.check           | CIDER + nREPL      | Primary  |
| Ruby/Rails         | Rails 7.2              | propcheck            | ruby/debug (rdbg)  | Done     |
| Python/FastAPI     | FastAPI + Starlette    | hypothesis           | debugpy            | Planned  |
| TypeScript/Next.js | Next.js 14 + React     | fast-check           | Node --inspect     | This doc |
| Go/net-http        | net/http + chi         | rapid                | delve (dlv)        | Planned  |

The Docker Compose shape is identical across stacks: app + postgres + redis +
kafka + otel-collector + jaeger + mailpit. Only the =web= service image,
debugger port protocol, PBT runner command, and framework-specific hooks change.
The Claude Code harness pattern (Bash + MCP chrome + hooks engine) is unchanged.

import { OTLPTraceExporter } from "@opentelemetry/exporter-trace-otlp-http";
import { HttpInstrumentation } from "@opentelemetry/instrumentation-http";
import { PgInstrumentation } from "@opentelemetry/instrumentation-pg";
import { IORedisInstrumentation } from "@opentelemetry/instrumentation-ioredis";

import { ATTR_SERVICE_NAME } from "@opentelemetry/semantic-conventions";

const sdk = new NodeSDK({
  resource: new Resource({
    [ATTR_SERVICE_NAME]: process.env.OTEL_SERVICE_NAME ?? "trace-spine-nextjs",
  }),
  traceExporter: new OTLPTraceExporter({
    url: process.env.OTEL_EXPORTER_OTLP_ENDPOINT

      : "http://localhost:4318/v1/traces",
  }),
  instrumentations: [
    new HttpInstrumentation(),
    new PgInstrumentation(),
    new IORedisInstrumentation(),
  ],
});

sdk.start();

** Middleware: trace-spine extraction/injection

#+begin_src typescript
// middleware.ts  (Next.js middleware — runs on every request)
import { NextRequest, NextResponse } from "next/server";
import { TraceSpine } from "trace-spine-ts";

const traceSpine = new TraceSpine({
  ingress: process.env.TRACE_SPINE_INGRESS === "true",
  serviceName: process.env.OTEL_SERVICE_NAME ?? "unknown",
});

export function middleware(request: NextRequest) {
  const traceparent = request.headers.get("traceparent");
  const result = traceSpine.continueOrStart(traceparent);

  const requestHeaders = new Headers(request.headers);
  requestHeaders.set("traceparent", result.traceparent);

  const response = NextResponse.next({
    request: { headers: requestHeaders },
  });

  // Propagate traceparent on the response for downstream consumers
  response.headers.set("traceparent", result.traceparent);

  return response;
}

export const config = {
  matcher: [
    // Match all request paths except static files and _next internals
    "/((?!_next/static|_next/image|favicon.ico).*)",
  ],
};
#+end_src

** Structured logging with pino

#+begin_src typescript
// lib/logger.ts
import pino from "pino";
import { trace, context } from "@opentelemetry/api";

export const logger = pino({
  level: process.env.LOG_LEVEL ?? "info",
  transport:
    process.env.NODE_ENV === "development"
      ? { target: "pino-pretty" }
      : undefined,
  mixin() {
    const span = trace.getSpan(context.active());
    if (span) {
      const spanContext = span.spanContext();
      return {
        trace_id: spanContext.traceId,
        span_id: spanContext.spanId,
        service: process.env.OTEL_SERVICE_NAME ?? "unknown",
      };
    }
    return {};
  },
});

*** /loop 5m — Log scan for invariant violations

#+begin_src bash
#!/usr/bin/env bash
# bin/dx-log-scan-docker — Scan Next.js container logs for I-spine-* violations
set -euo pipefail

LOGS=$(docker compose logs web --since 5m --no-log-prefix 2>/dev/null)
VIOLATIONS=0

# I-spine-present: missing traceparent
COUNT=$(echo "$LOGS" | grep -ci "traceparent.*missing\|no.traceparent\|TraceSpine.*MissingContext" || true)
if [[ "$COUNT" -gt 0 ]]; then
  echo "VIOLATION I-spine-present: $COUNT occurrence(s) in last 5m"
  VIOLATIONS=$((VIOLATIONS + COUNT))
fi

# I-spine-wellformed: malformed header
COUNT=$(echo "$LOGS" | grep -ci "malformed.traceparent\|TraceSpine.*ParseError" || true)
if [[ "$COUNT" -gt 0 ]]; then
  echo "VIOLATION I-spine-wellformed: $COUNT occurrence(s)"
  VIOLATIONS=$((VIOLATIONS + COUNT))
fi

# I-spine-no-fabrication: internal origination
COUNT=$(echo "$LOGS" | grep -ci "ProgrammerError.*originate\|TraceSpine.*FabricationError" || true)
if [[ "$COUNT" -gt 0 ]]; then
  echo "VIOLATION I-spine-no-fabrication: $COUNT occurrence(s)"
  VIOLATIONS=$((VIOLATIONS + COUNT))
fi

# I-spine-no-leak: trace ID in response body
COUNT=$(echo "$LOGS" | grep -cE '"trace_id":"[0-9a-f]{32}".*response_body' || true)
if [[ "$COUNT" -gt 0 ]]; then
  echo "VIOLATION I-spine-no-leak: $COUNT occurrence(s)"
  VIOLATIONS=$((VIOLATIONS + COUNT))
fi

# Budget: slow propagation
COUNT=$(echo "$LOGS" | grep -ci "TraceSpine.*BudgetExceeded\|propagation.*exceeded.*budget" || true)
if [[ "$COUNT" -gt 0 ]]; then
  echo "VIOLATION I-spine-budget: $COUNT occurrence(s)"
  VIOLATIONS=$((VIOLATIONS + COUNT))
fi

# Summary
if [[ $VIOLATIONS -gt 0 ]]; then
  echo "---"
  echo "TOTAL: $VIOLATIONS violation(s) in last 5m"
  echo "View traces: http://localhost:16686"
  exit 1
fi

echo "No violations in last 5m"
#+end_src

*** /loop 30m — Trace continuity sampling

#+begin_src bash
#!/usr/bin/env bash
# bin/dx-trace-continuity — Sample recent traces from Jaeger for discontinuity
set -euo pipefail

SERVICE="${OTEL_SERVICE_NAME:-checkout}"
JAEGER="http://localhost:16686"

# Fetch recent traces via Jaeger API
TRACES=$(curl -sf "$JAEGER/api/traces?service=$SERVICE&limit=10&lookback=30m" \
  | jq -r '.data[]' 2>/dev/null)

if [[ -z "$TRACES" ]]; then
  echo "No traces found in last 30m (is the app receiving traffic?)"
  exit 0
fi

# Check each trace for discontinuity
DISCONTINUITIES=0
TRACE_COUNT=$(echo "$TRACES" | jq -s 'length')

for i in $(seq 0 $((TRACE_COUNT - 1))); do
  TRACE=$(echo "$TRACES" | jq -s ".[$i]")
  TRACE_ID=$(echo "$TRACE" | jq -r '.traceID')
  SPAN_COUNT=$(echo "$TRACE" | jq '.spans | length')
  UNIQUE_TRACES=$(echo "$TRACE" | jq '[.spans[].traceID] | unique | length')

  if [[ "$UNIQUE_TRACES" -gt 1 ]]; then
    echo "DISCONTINUITY: trace $TRACE_ID has $UNIQUE_TRACES different trace IDs across $SPAN_COUNT spans"
    DISCONTINUITIES=$((DISCONTINUITIES + 1))
  fi
done

echo "Sampled $TRACE_COUNT traces: $DISCONTINUITIES discontinuity(s)"
if [[ $DISCONTINUITIES -gt 0 ]]; then
  echo "View: $JAEGER/search?service=$SERVICE"
  exit 1
fi

#+begin_src bash
#!/usr/bin/env bash
# bin/dx-migration-check — Detect pending migrations or schema drift
set -euo pipefail

STATUS=$(docker compose exec -T web pnpm prisma migrate status 2>/dev/null)

PENDING=$(echo "$STATUS" | grep -c "Following migration.*not yet been applied" || true)
if [[ "$PENDING" -gt 0 ]]; then
  echo "PENDING MIGRATIONS:"
  echo "$STATUS" | grep "not yet been applied"
  echo "Run: docker compose exec web pnpm prisma migrate deploy"
  exit 1
fi

# Check prisma schema is committed and matches
SCHEMA_DIFF=$(docker compose exec -T web git diff --stat prisma/schema.prisma 2>/dev/null || true)
if [[ -n "$SCHEMA_DIFF" ]]; then
  echo "SCHEMA DRIFT: prisma/schema.prisma has uncommitted changes"
  echo "$SCHEMA_DIFF"
  exit 1
fi

echo "Database: migrations applied, none pending"

** Claude Code + Playwright via MCP chrome

The =claude-in-chrome= MCP server gives Claude Code browser automation
equivalent to Playwright, operating against the live Next.js app:

*** Trace verification through the browser

#+begin_src text
Scenario: Verify checkout flow carries traceparent end-to-end

1. Claude Code → mcp__claude-in-chrome__navigate
   URL: http://localhost:3000/products

2. Claude Code → mcp__claude-in-chrome__find
   Find: "Add to Cart" button → click

3. Claude Code → mcp__claude-in-chrome__navigate
   URL: http://localhost:3000/checkout

4. Claude Code → mcp__claude-in-chrome__read_network_requests
   Filter: XHR to /api/checkout
   Assert: request headers contain "traceparent" matching
           ^00-[0-9a-f]{32}-[0-9a-f]{16}-[0-9a-f]{2}$

5. Claude Code → Bash
   curl -sf "http://localhost:16686/api/traces?service=checkout&limit=1"
   Assert: trace spans form connected DAG, single trace_id throughout

6. Claude Code → mcp__claude-in-chrome__read_console_messages
   Pattern: "TraceSpine"
   Assert: no error-level messages
#+end_src

   Start recording: "checkout-flow-trace-verify.gif"

   Capture extra frames at each step transition

*** DOM inspection for I-spine-no-leak

#+begin_src text
Scenario: Verify trace IDs don't leak to rendered HTML

1. Claude Code → mcp__claude-in-chrome__navigate
   URL: http://localhost:3000/orders/123

2. Claude Code → mcp__claude-in-chrome__get_page_text
   Search page text for 32-character hex strings

3. Claude Code → mcp__claude-in-chrome__javascript_tool
   Script: document.body.innerHTML.match(/[0-9a-f]{32}/g)
   Assert: no matches (trace IDs must not appear in DOM)

4. If violation found:
   Claude Code → Bash
   bd create --title="I-spine-no-leak: trace ID in order detail page" \
     --type=bug --priority=1
#+end_src

** Debugger integration

Claude Code can connect to the Node.js inspector when investigating
invariant violations discovered by hooks:

#+begin_src text
Scenario: Hook detects I-spine-no-fabrication, need to inspect middleware

1. Log scan hook reports: "VIOLATION I-spine-no-fabrication"

2. Claude Code reads the violation log to identify the route/handler

3. Claude Code → Bash
   # Add a debugger statement to the trace-spine middleware
   # In middleware.ts, add: debugger; before the continueOrStart call

4. Developer triggers the request via browser or curl

5. Claude Code → mcp__claude-in-chrome__navigate
   URL: chrome://inspect/#devices
   # Node process at localhost:9229 will show as paused
   # Inspect: traceparent header value, request.headers, ingress config
   # Determine: why continueOrStart raised FabricationError

6. Claude Code reports findings and suggests fix
#+end_src

** Required Claude Code configuration

For the harness to work, Claude Code needs:

| Requirement                  | How to configure                                   |
|------------------------------+----------------------------------------------------|
| Chrome MCP tools             | =claude-in-chrome= extension installed in Chrome   |
| Bash access to docker        | Docker Desktop running; =docker compose= on PATH   |
| Node.js inspector access     | Port 9229 exposed; =chrome://inspect= configured   |
| Jaeger API access            | Port 16686 exposed in docker-compose               |
| PostgreSQL client            | =psql= on PATH (=brew install libpq=)              |
| Redis client                 | =redis-cli= on PATH (=brew install redis=)         |
| Kafka client                 | =kcat= on PATH (=brew install kcat=)               |
| beads                        | =bd= on PATH                                       |
| Session hooks                | =.claude/settings.json= configured per this spec   |
