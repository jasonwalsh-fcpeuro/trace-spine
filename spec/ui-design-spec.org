* Overview

This spec defines the three UI surfaces in the Petstore Commerce Platform.
All UI containers are ClojureScript + Re-frame SPAs that must inject W3C
=traceparent= on every outbound request to the API Gateway.

* Shared constraints

** Trace propagation (mandatory)

Every UI surface must:

1. Generate a =traceparent= header for each XHR/fetch to the API Gateway
2. Use the =lib/cljs= adapter (=trace-spine.cljs.core=)
3. Call =continue-or-start= with =is-ingress: true= (browser is the ingress boundary)
4. Pass trace context through Re-frame's interceptor chain
5. Include =traceparent= in all =re-frame.http-fx= requests

#+begin_src clojure
;; Re-frame interceptor for trace propagation
(def trace-interceptor
  (re-frame/->interceptor
    :id    :trace-spine
    :before (fn [context]
              (let [ctx (trace-spine/continue-or-start nil true)]
                (assoc-in context [:coeffects :trace-context] ctx)))))

;; HTTP effect handler wraps all outbound calls
(reg-fx :http-xhrio
  (fn [request]
    (let [ctx (subscribe [:trace-context])]
      (-> request
          (assoc-in [:headers "traceparent"] (trace-spine/format ctx))
          (ajax/send)))))
#+end_src

** Technology stack

| Layer      | Choice                | Rationale                                   |
|------------+-----------------------+---------------------------------------------|
| Language   | ClojureScript 1.12    | Same substrate as backend; shared contracts |
| Framework  | Re-frame              | Predictable state; interceptor chain        |
| Build      | shadow-cljs           | npm interop; hot reload                     |
| CSS        | Tailwind CSS          | Utility-first; no custom design system yet  |
| Components | Headless UI (reagent) | Accessible primitives; style-agnostic       |
| Router     | reitit-frontend       | Consistent with API Gateway routing         |
| HTTP       | re-frame-http-fx      | Effect-driven; trace interceptor compatible |
| Testing    | cljs.test + devcards  | Visual component testing                    |

** Shared layout principles

- Responsive: mobile-first, breakpoints at =sm= =md= =lg= =xl=
- Accessible: WCAG 2.1 AA minimum; keyboard navigation; screen reader labels
- Performance: initial bundle < 200KB gzipped; lazy-load non-critical routes
- Error states: every async operation has loading / success / error states
- Empty states: every list view has an empty state with guidance

*** Trace contract for checkout

The checkout flow is the highest-value trace in the system:

#+begin_src text
Browser (traceparent created)
  → API Gateway (extract, child span)
    → Fraud Service (score request)
    → Order Service (create order)
      → Payment Service (charge)
        → Stripe (external; trace ends)
      → Kafka (order.created event)
        → WMS (fulfill order)
#+end_src

Every step in this chain must share the same =trace-id=. The storefront
SPA is responsible for originating the trace and attaching it to the initial
=/checkout= POST.
